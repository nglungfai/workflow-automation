Getting Started with Claude Cowork

Getting Started · 6 min read

Cowork is Claude's mode for working directly on your files and tasks — not just chatting about them. If you've used Claude for research or drafting before, the mental shift is: Cowork can read your folders, write and edit real files, run code in a sandbox, and hand you back finished deliverables (docs, spreadsheets, decks) rather than pasted text.

How Cowork is different from a normal chat

In a regular chat, you get an answer in the conversation. In Cowork, Claude can:

  • Read files from a folder you connect (a shared drive, a project folder, an export from a tool)
  • Create and edit real files — Word docs, Excel workbooks, PowerPoint decks, PDFs, code — saved to a folder you can open directly
  • Run commands in an isolated sandbox (useful for data cleaning, quick scripts, or checking a calculation)
  • Use connected tools (Slack, Jira/Confluence, Google Drive, and internal MCP connectors like our Predactiv audience platform) to pull live data instead of you copy-pasting it in
  • Follow multi-step task lists and keep working through them, rather than requiring a single one-shot prompt

The first three things to try

1. Point it at a real folder

Connect a folder you actually work in — a project directory, a shared drive folder — and ask Cowork to summarize what's in it, or to clean up a messy spreadsheet you already have. Seeing it operate on your own files (not a toy example) is the fastest way to build intuition for what it can do.

2. Ask for a finished artifact, not just an answer

Instead of "what should our POC status update say," try "draft a POC status update as a Word doc I can send to the partner." Cowork will produce the actual file. This is the biggest behavior change from chat-only usage — think in terms of deliverables.

3. Let it use a task list for anything with more than two steps

For multi-step requests (research, then build a doc, then check it), Cowork will often show a visible task list as it works. You don't need to ask for this explicitly — but if a request feels big, breaking it into a numbered list yourself in your prompt helps it plan the work the way you want.

Where it fits into an AI transformation effort

The pitch internally isn't "AI replaces X." It's narrower and more useful: Cowork removes the busywork around drafting, formatting, and first-pass synthesis so the person can spend their judgment where it actually matters — reviewing, deciding, and refining. For a data-heavy org like ours, that shows up in status reporting, documentation, audience/data QA passes, and technical write-ups for partners.
