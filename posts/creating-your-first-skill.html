Creating Your First Skill: A Beginner's Guide

Skill Deep Dive · 4 min read

Half the posts on this site mention a skill by name — predactiv-design, agent-new-syndicated-segments, agent-pm — but none of them explain what a skill actually is or how to build one from scratch. If you've been installing other people's skills and wondering how they're made, this is that post.

What a skill actually is

A skill is a folder. That's the whole starting point. Inside that folder sits one required file, SKILL.md, plus whatever else you want to bundle alongside it — reference docs, scripts, templates. Anthropic's own definition: Agent Skills are modular capabilities that extend Claude's functionality, each one packaging instructions, metadata, and optional resources that Claude uses automatically when relevant. Nothing about that requires special tooling — SKILL.md is a plain markdown file with a small YAML header on top.

The reason skills exist instead of just re-explaining things in your prompt every time: a skill loads on demand and eliminates the need to repeat the same guidance across conversations. Write the instructions once, install the skill, and every future request that matches its description pulls those instructions in automatically — no "remember how we do this" reminder needed.

The one thing that makes skills different from a saved prompt: progressive disclosure

This is the part beginners usually skip past, and it's the part that actually explains why skills scale where a giant system prompt doesn't. Skill content loads in three stages, and Claude only pays the token cost for what it actually needs:

Practically, the description field is doing real work: it's the only thing Claude sees before deciding whether your skill is even relevant. Front-load concrete "use this when..." trigger phrases — a vague description means your skill either never fires or fires on the wrong requests.

Anatomy of a SKILL.md

Two required frontmatter fields: name (lowercase letters, numbers, hyphens only, max 64 characters, can't contain "anthropic" or "claude") and description (max 1,024 characters, can't be empty — and it's the field that decides whether your skill triggers). Below that, the body is just markdown instructions, written the way you'd brief a new hire:

---
name: weekly-metrics-digest
description: Pull last week's key product metrics into a short digest. Use when the user asks for "weekly numbers", "metrics recap", or "how did we do last week".
---

# Weekly Metrics Digest
1. Pull signups, activation rate, and churn for the last 7 days.
2. Compare against the prior 7 days, call out any move greater than 10%.
3. Write 3-5 sentences, ending with one recommended action.

That's already a complete, installable skill — no scripts or bundled files required. Reach for Level 3 resources only once instructions alone get unwieldy, or you need something deterministic like a script that formats a spreadsheet the same way every time.

Building one, step by step

  1. Start from a real, narrow problem. "Summarize meetings" is too broad — Cowork already does that unprompted. "Turn our Monday standup notes into the yesterday/today/blockers format our CPO expects" is narrow enough to be worth packaging.
  2. Draft and test it. Use the skill-creator skill (also installed here at Predactiv) to scaffold the frontmatter and check your description against the rules above. Then paste the draft into a real conversation and try the exact prompts teammates would use — if it doesn't trigger, the description is almost always the culprit.
  3. Package and install. Zip the skill's folder so the zip contains the wrapping folder itself (matching name:) with SKILL.md inside — not a flat dump of loose files. Then upload the .skill file under Settings → Capabilities → Skills. From then on, matching requests trigger it automatically.

Try it

Pick one thing you re-explain to Cowork more than twice a month — a report format, a checklist, a way you always want data pulled — and ask Cowork to help you build it:

"Use skill-creator to help me draft a skill that turns raw survey export CSVs into a themed summary with a top-3-issues callout, formatted the way I do it in Confluence today."

Skill-creator will scaffold the frontmatter, push back if your description is too vague to trigger reliably, and can run a quick eval against your draft before you package and install it. One rule worth carrying forward: only install skills from people or sources you trust — a skill's instructions can direct Claude to run code and access files, so treat installing one the way you'd treat installing any other piece of software on a work machine.
