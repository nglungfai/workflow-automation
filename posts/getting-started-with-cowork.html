Getting Started with Claude Cowork

Getting Started · 7 min read

Cowork is Claude's mode for working directly on your files and tasks — not just chatting about them. If you've used Claude for research or drafting before, the mental shift is: Cowork can read your folders, write and edit real files, run code in a sandbox, and hand you back finished deliverables (docs, spreadsheets, decks) rather than pasted text.

Your first 5 minutes. If you have Google Drive for Desktop, your Drive files are already a folder Cowork can connect to — no export needed. Connect a project folder from My Drive or a shared drive, then paste this:

Try: "Look through this folder and tell me what's here, which files were touched most recently, and where the latest project left off."

Nothing gets modified — Cowork reads and reports back. Product team: connect the Atlassian connector instead and try "Summarize the last three updates to our team's Confluence space and flag any open decisions."

How Cowork is different from a normal chat

In a regular chat, you get an answer in the conversation. In Cowork, Claude can:

  • Read files from a folder you connect (a shared drive, a project folder, an export from a tool)
  • Create and edit real files — Word docs, Excel workbooks, PowerPoint decks, PDFs, code — saved to a folder you can open directly
  • Run commands in an isolated sandbox (useful for data cleaning, quick scripts, or checking a calculation)
  • Use pre-built connectors (Slack, Confluence, Google Drive, and our own Predactiv audience platform) to pull live data instead of you copy-pasting it in
  • Follow multi-step task lists and keep working through them, rather than requiring a single one-shot prompt

Three habits that make Cowork click

1. Go deeper with the folder you connected

If you ran the five-minute prompt above, Cowork has already read your files. The next step is asking for something that produces a file: point it at a messy tracker and ask for a cleaned copy, or have it turn a folder of scattered notes into one organized summary doc. Working on your own files (not a toy example) is the fastest way to build intuition for what it can do.

2. Ask for a finished artifact, not just an answer

Instead of "what should our POC status update say," try "draft a POC status update as a Word doc I can send to the partner." Cowork will produce the actual file. This is the biggest behavior change from chat-only usage — think in terms of deliverables.

3. Let it use a task list for anything with more than two steps

For multi-step requests (research, then build a doc, then check it), Cowork will often show a visible task list as it works. You don't need to ask for this explicitly — but if a request feels big, breaking it into a numbered list yourself in your prompt helps it plan the work the way you want.

Common hesitations

"Will it change my files without asking?" No. Reading is just reading — the five-minute prompt above modifies nothing. When you do ask Cowork to edit or create files, it works through the changes visibly and asks before anything destructive.

"Is my data safe? Where does code actually run?" Scripts and commands run in an isolated sandbox, not loose on your machine, and only the folders you explicitly connect are visible to it. Connect a narrow project folder to start, not your whole drive.

"Do I need IT to set something up?" No. If you have the Claude desktop app and Google Drive for Desktop, you're ready — connecting a folder is the whole setup. Connectors like Confluence are one authorization click, and everyone at Predactiv already has access.
